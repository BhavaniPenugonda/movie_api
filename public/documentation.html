API Documentation

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints Table

Request URL Response
Return a list of ALL movies GET /movies Returns a list of all movies available in the database.
Return data about a single movie by title GET /movies/title/{title} Returns detailed data (description, genre, director, image URL, featured status) about a specific movie by its title.
Return data about a genre by name/title GET /genres/{genre_name} Returns data (description) about a specific genre by its name/title.
Return data about a director by name GET /directors/{director_name} Returns data (bio, birth year, death year) about a specific director by name.
Allow new users to register POST /users Creates a new user account. Returns the details of the newly registered user.
Allow users to update their user info (username) PUT /users/{user_id} Updates the username of the specified user. Returns the updated user details.
Allow users to add a movie to their list of favorites POST /users/{user_id}/favorites/{movie_id} Adds a movie to the user's favorites list. Returns a confirmation message.
Allow users to remove a movie from their list of favorites DELETE /users/{user_id}/favorites/{movie_id} Removes a movie from the user's favorites list. Returns a confirmation message.
Allow existing users to deregister DELETE /users/{user_id} Deletes the user account. Returns a confirmation message that the user has been deregistered.

Endpoints

1. Return a list of ALL movies

Description: Retrieves a list of all movies available in the database.

URL: /movies

Method: GET

Query Parameters: None

Request Body: None

Response Format: JSON array of movie objects


    [
    {
      "id": "1",
      "title": "Inception",
      "description": "A mind-bending thriller...",
      "genre": "Thriller",
      "director": "Christopher Nolan",
      "imageUrl": "http://example.com/inception.jpg",
      "featured": true
    },
    ...
    ]

2. Return data about a single movie by title

Description: Retrieves detailed information about a specific movie by its title.

URL: /movies/title/{title}

Method: GET

Query Parameters: title (The title of the movie)

Request Body: None

Response Format: JSON object with movie details


    {
      "id": "1",
      "title": "Inception",
      "description": "A mind-bending thriller...",
      "genre": "Thriller",
      "director": "Christopher Nolan",
      "imageUrl": "http://example.com/inception.jpg",
      "featured": true
    }

3. Return data about a genre by name/title

Description: Retrieves the description of a specific genre by its name.

URL: /genres/{genre_name}

Method: GET

Query Parameters: genre_name (The name of the genre)

Request Body: None

Response Format: JSON object with genre details


    {
      "name": "Thriller",
      "description": "A genre characterized by excitement and suspense..."
    }

4. Return data about a director by name

Description: Retrieves the biography and life details of a specific director by their name.

URL: /directors/{director_name}

Method: GET

Query Parameters: director_name (The name of the director)

Request Body: None

Response Format: JSON object with director details


    {
      "name": "Christopher Nolan",
      "bio": "An acclaimed director known for his innovative films...",
      "birthYear": "1970",
      "deathYear": null
    }

5. Allow new users to register

Description: Creates a new user account in the system.

URL: /users

Method: POST

Query Parameters: None

Request Body: JSON object with user details


    {
      "username": "johndoe",
      "password": "password123",
      "email": "john.doe@example.com"
    }

Response Format: JSON object with the details of the newly registered user


    {
      "id": "1",
      "username": "johndoe",
      "email": "john.doe@example.com"
    }

6. Allow users to update their user info (username)

Description: Updates the username of an existing user.

URL: /users/{user_id}

Method: PUT

Query Parameters: user_id (The ID of the user)

Request Body: JSON object with the new username


    {
      "username": "newusername"
    }

Response Format: JSON object with the updated user details


    {
      "id": "1",
      "username": "newusername",
      "email": "john.doe@example.com"
    }

7. Allow users to add a movie to their list of favorites

Description: Adds a movie to the user's list of favorite movies.

URL: /users/{user_id}/favorites/{movie_id}

Method: POST

Query Parameters: user_id (The ID of the user), movie_id (The ID of the movie)

Request Body: None

Response Format: JSON object confirming the addition of the movie


    {
      "message": "Movie has been added to favorites."
    }

8. Allow users to remove a movie from their list of favorites

Description: Removes a movie from the user's list of favorite movies.

URL: /users/{user_id}/favorites/{movie_id}

Method: DELETE

Query Parameters: user_id (The ID of the user), movie_id (The ID of the movie)

Request Body: None

Response Format: JSON object confirming the removal of the movie


    {
      "message": "Movie has been removed from favorites."
    }

9. Allow existing users to deregister

Description: Deletes the user account from the system.

URL: /users/{user_id}

Method: DELETE

Query Parameters: user_id (The ID of the user)

Request Body: None

Response Format: JSON object confirming the deletion of the user account


    {
      "message": "User has been deregistered."
    }
